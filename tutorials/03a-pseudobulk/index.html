Pseudobulk differential expression across AD pathology

Donors as replicates: aggregating single cells and testing with edgeR

Author

IPRE Winter 2026 · MOTCO · research line IPI-26-1206

Tutorials 01 and 02 stayed inside a single donor: enough to learn the object and the preprocessing steps, but not enough to answer the question the SEA-AD project actually cares about—which genes change with Alzheimer’s pathology? That is a comparison between people, so this tutorial is the first that loads several donors at once.

The temptation, once you have hundreds of thousands of cells, is to run a differential-expression test cell by cell: pool all the astrocytes from low-pathology donors, all the astrocytes from high-pathology donors, and compare the two piles. That answer is statistically wrong, and understanding why is the main idea of this tutorial. The fix—pseudobulk—is simple once you see the problem: collapse each donor’s cells of a given type into one summed profile, and compare those donor-level profiles the way you would a classic bulk RNA-seq experiment.

Learning objectives

By the end of this tutorial you should be able to:

  • Explain why the donor is the unit of replication in a cross-condition comparison, and why per-cell tests commit pseudoreplication—treating non-independent cells as independent and massively inflating significance.
  • Load several donors and assemble a per-donor metadata table (pathology group, sex, age).
  • Aggregate single-cell counts into pseudobulk profiles per donor × cell type with aggregateAcrossCells.
  • Run a standard edgeR differential-expression pipeline—filtering, TMM normalization, dispersion estimation, and a quasi-likelihood test—on those profiles.
  • Read the results table and an MA/volcano plot critically, and list the confounds that make a small brain-cohort comparison hard.

Prerequisites: tutorials 00–02. Unlike the trajectory track, this one does not reuse the saved object from tutorial 02: pseudobulk works from raw counts (you sum counts, then let edgeR do its own normalization), and it needs many donors rather than one. You will, however, reuse the loading pattern from tutorial 01 and the QC logic from tutorial 02.

Expected outcome: a pseudobulk count matrix (genes × donor) for one cell type, an edgeR results table ranking genes by differential expression between low- and high-pathology donors, and a defensible read of what it does and does not show.

Step 2 — Why pseudobulk? The pseudoreplication trap

Imagine two groups of donors—low pathology and high pathology—and suppose you have 5,000 astrocytes from each group. A per-cell test (a t-test or a single-cell DE method) treats those 10,000 cells as 10,000 independent observations. They are not. Cells from the same donor share that donor’s genotype, age, medications, RNA quality, and dissection batch; they are far more alike than cells from different donors. Statistically, they are pseudoreplicates—repeated measurements of the same biological unit, not new biological units.

The consequence is severe. With thousands of correlated cells, the standard error shrinks toward zero and almost every gene comes out “significant,” even genes that differ only because of one unusual donor. The test is answering “are these two piles of cells different?”—which they always are—instead of “does pathology change expression across people?”, which is the question you meant to ask.

Pseudobulk restores the right unit of analysis. For each donor, sum the counts across all cells of one type into a single profile. Now each donor contributes one number per gene, and a cohort of, say, 12 donors gives 12 independent replicates—exactly the design that bulk RNA-seq methods like edgeR and limma-voom were built for. Summing (not averaging) keeps the data as integer counts, so the count-based noise models still apply, and it correctly gives more weight to donors and cell types with more cells.

NoteThis is now the field’s default

Early single-cell DE papers used per-cell tests and reported thousands of hits. Later re-analyses (Squair et al. 2021, Crowell et al. 2020) showed that pseudobulk with donor-level replication controls false positives far better. For a cross-condition comparison with biological replicates, pseudobulk is the default recommendation.

Step 3 — Choose your donors

Pseudobulk lives or dies by the cohort you assemble. Two rules:

  1. Contrast on pathology. You need donors spanning the pathology range so there is something to compare. SEA-AD grades each donor with Overall AD neuropathological Change (ADNC)—Not AD, Low, Intermediate, High—a standard summary of amyloid and tau burden. We will collapse this into two groups: low (Not AD, Low) versus high (Intermediate, High).
  2. Balance the confounds. Any variable that differs systematically between your groups can masquerade as a pathology effect. Sex and age are the usual suspects in a brain cohort; try to keep them comparable across groups, and we will also put sex in the model in Step 7.

Download the donor files below from the SEA-AD data browser exactly as in tutorial 01 (PFC → RNAseq → donors_objects), and save each into the shared data/sea-ad/ folder named donor_<ID>.h5ad—for example data/sea-ad/donor_H20.33.001.h5ad. You already have the first one from tutorial 01.

TipWhy these donors, and how many?

This is a deliberately small, balanced set chosen to run in a class session—a real study would use every available donor. More donors mean more statistical power and less sensitivity to any single unusual brain. The donor is your replicate, so donor count, not cell count, is what limits this analysis: going from 6 to 60 donors helps far more than sequencing more cells per donor.

Step 4 — Load and lightly QC each donor

We load one donor at a time, do the same count-based QC as tutorial 02, and immediately collapse it to pseudobulk—so only one full-size donor object is ever in memory. The function below wraps the tutorial-01 loading pattern and the tutorial-02 QC into a single reusable step.

library(schard)
library(SingleCellExperiment)
library(scuttle)

data_dir <- "../../data/sea-ad"

load_donor <- function(id) {
  h5 <- file.path(data_dir, paste0("donor_", id, ".h5ad"))

  # Load the object and swap in the raw UMI counts (see tutorial 02, Step 2).
  sce <- schard::h5ad2sce(h5)
  counts_mat <- schard::h5ad2Matrix(h5, "layers/UMIs")
  dimnames(counts_mat) <- dimnames(sce)
  counts(sce) <- counts_mat
  assay(sce, "X") <- NULL

  # Count-based QC: drop low-quality nuclei (tutorial 02, Steps 3–4).
  is_mito <- grepl("^MT-", rownames(sce))
  sce <- addPerCellQCMetrics(sce, subsets = list(Mito = is_mito))
  discard <- perCellQCFilters(sce, sub.fields = "subsets_Mito_percent")$discard
  sce <- sce[, !discard]

  sce
}

Step 5 — Aggregate to pseudobulk

aggregateAcrossCells (from scuttle) sums the count matrix within groups of cells. We group by cell type (Subclass); because we process one donor at a time, every cell already belongs to the same donor, so each donor contributes one pseudobulk profile per cell type. The donor-level columns (Donor ID, ADNC, Sex) are constant within the donor, so they carry through unchanged, and the result records how many cells went into each sum (ncells).

pseudobulk_one <- function(sce) {
  aggregateAcrossCells(
    sce,
    ids = sce$Subclass,
    use.assay.type = "counts",
    statistics = "sum"
  )
}

# Loop over donors, collapsing each to pseudobulk before moving on.
pb_list <- lapply(donor_ids, function(id) {
  message("Aggregating ", id)
  pseudobulk_one(load_donor(id))
})

# Combine into one object: rows = genes, columns = donor × cell type.
pb <- do.call(cbind, pb_list)
pb

Each column of pb is one donor’s summed counts for one cell type. Attach a tidy pathology group and confirm the layout.

# Two-group pathology label from ADNC.
adnc <- pb$`Overall AD neuropathological Change`
pb$pathology <- factor(
  ifelse(adnc %in% c("Not AD", "Low"), "low", "high"),
  levels = c("low", "high")
)

# Cell type per column comes from the aggregation key.
pb$cell_type <- pb$ids

# How many donors per pathology group, and cells behind each pseudobulk column?
table(pb$cell_type, pb$pathology)
summary(pb$ncells)
ImportantAggregate raw counts, not normalized values

Pseudobulk sums counts, which is why tutorial 02’s normalized logcounts are the wrong input here—you cannot meaningfully sum log-normalized values, and edgeR needs integer counts to model. We deliberately reloaded the raw UMIs layer in load_donor for exactly this reason.

library(edgeR)

y <- DGEList(
  counts  = assay(pb_ct, "counts"),
  samples = as.data.frame(colData(pb_ct)),
  group   = pb_ct$pathology
)
y$genes <- data.frame(gene = rownames(pb_ct))

Step 7 — Filter, normalize, and test

Three standard edgeR steps precede the test:

  • Filter genes that are lowly expressed across the board—they carry no information and only cost you multiple-testing power. filterByExpr picks a sensible expression floor from the group sizes.
  • Normalize for composition with TMM (calcNormFactors). Even after summing, one donor can have a few genes dominating its library; TMM corrects for that so a handful of high genes don’t make everything else look down.
  • Model and test. We fit a negative-binomial model with a quasi-likelihood F-test (glmQLFit/glmQLFTest), the recommended edgeR workflow for designed experiments with replicates.
keep <- filterByExpr(y, group = y$samples$pathology)
y <- y[keep, , keep.lib.sizes = FALSE]
y <- calcNormFactors(y)

# Model pathology, adjusting for sex so a sex imbalance can't masquerade as a
# pathology effect. coef = "pathologyhigh" is the high-vs-low contrast.
design <- model.matrix(~ Sex + pathology, data = y$samples)

y <- estimateDisp(y, design)
fit <- glmQLFit(y, design)
res <- glmQLFTest(fit, coef = "pathologyhigh")
NoteIf your donors are all one sex

~ Sex + pathology only makes sense if both sexes appear in your set. If you picked donors of a single sex, drop that term and use design <- model.matrix(~ pathology, data = y$samples). Balancing sex is better than ignoring it—but with a handful of donors you can’t always have both.

Step 8 — Read the results

topTags ranks genes by evidence for a pathology effect; decideTests summarizes how many pass a false-discovery-rate cutoff.

topTags(res, n = 20)

# Up / not-significant / down at FDR < 0.05:
summary(decideTests(res))

The columns that matter: logFC is the log2 fold change (high vs low pathology; positive means higher in high-pathology donors), and FDR is the Benjamini–Hochberg-adjusted p-value—your significance yardstick once you are testing thousands of genes. A mean–difference (MA) plot shows the whole gene set at once, with the significant genes highlighted.

plotMD(res, main = paste("High vs low pathology —", focus))
abline(h = 0, col = "grey", lty = 2)
TipOptional: a volcano plot

The MA plot puts fold change against expression level; a volcano plot puts fold change against significance, which some people find easier to read. Build one from the same results table:

tt <- topTags(res, n = Inf)$table
plot(tt$logFC, -log10(tt$FDR),
     xlab = "log2 fold change (high vs low)",
     ylab = "-log10 FDR", pch = 16, cex = 0.4,
     col = ifelse(tt$FDR < 0.05, "firebrick", "grey60"))
abline(h = -log10(0.05), lty = 2)

Step 9 — Interpret, with the caveats up front

A results table is easy to over-read. Before believing any gene:

  • Power is set by donor count. With only a handful of donors per group, you can detect only large, consistent effects; a short or empty hit list does not mean “nothing changes,” it means this cohort can’t resolve it. This is the single biggest limitation of a class-sized run.
  • Confounds. We adjusted for sex, but age, post-mortem interval, RNA quality (RIN), and dissection batch can all track with pathology. A real analysis checks these explicitly.
  • Cell-type assignment. Pseudobulk inherits the Subclass labels; if cells were mislabeled or a type is contaminated by ambient RNA, that noise flows into the profile.
  • Direction is relative to your grouping. logFC > 0 means higher in the high group only because of how we coded pathology; always re-check the contrast before writing a sentence about biology.

Treat the ranked list as hypotheses. The honest next step is to see whether the top genes overlap known AD biology and whether they replicate in an independent cohort.

Where this connects

The other analysis track, trajectory, asks a different question of the same data—ordering cells along a continuous process rather than comparing groups of donors. Both start from SEA-AD single cells; they diverge in what they treat as the unit of analysis. Pseudobulk’s unit is the donor; trajectory’s is the individual cell.

If something breaks

Warningcbind fails: rows/genes don’t match

do.call(cbind, pb_list) needs every donor to have the same genes in the same order. If a donor’s file has a different feature set, intersect to the common genes first: compute common <- Reduce(intersect, lapply(pb_list, rownames)) and subset each element to common before combining.

WarningA pathology group has only one donor

edgeR cannot estimate variability from a single replicate. Check table(pb$cell_type, pb$pathology) and add donors until both groups have at least two—ideally three or more—in your focal cell type.

WarningfilterByExpr removes almost every gene

This usually means the pseudobulk libraries are tiny—your focal cell type is rare, so each donor’s sum is small. Confirm with summary(pb_ct$ncells) and either raise the ncells cutoff and re-examine, or choose a more abundant cell type.

WarningaggregateAcrossCells can’t find Subclass

Column names must match exactly. Reprint them with colnames(colData(one)) and copy the label as shown (see tutorial 01 on backticks for names with spaces).
